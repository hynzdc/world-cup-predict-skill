# World Cup Predict Skill

一个面向 Codex 的世界杯 / 足球押注分析 Skill。

它的目标不是“保证命中”，而是把每天的足球截图处理成一套可复用流程：识别比赛、查询赔率和球队信息、做风险控制、输出店主能看懂的下单内容，并把每日投注和赛后盈亏记录成 Markdown。

> 说明：本项目只做比赛分析、记录和风险控制辅助，不构成投资建议，也不保证盈利。体育竞猜波动很大，请遵守当地法律法规，并严格控制预算。

## 项目能做什么

- 识别用户发来的比赛截图，提取日期、场次、开球时间、球队、胜平负概率和预测比分。
- 结合最新赔率、球队新闻、伤停、近期状态、旅途休整、公开采访和精神状态信号做赛前分析。
- 输出三类常用单关投注单：
  - 比分单
  - 胜平负 / 让球胜平负单
  - 总进球数单
- 生成“店主能看懂”的下单表格。
- 创建或更新每日 Markdown 记录。
- 比赛结束后，根据结果截图更新命中、回报和盈亏。
- 长期按玩法统计 ROI，帮助减少亏损较大的投注类型。

## 核心流程

### 1. 赛前预测

用户发一张比赛截图后，Skill 会要求 Codex 做这些事：

1. 从截图里提取比赛日期、场次、球队、时间、场地、可见概率和预测比分。
2. 上网查询最新信息，因为赔率、伤停和阵容会变。
3. 重点查询：
   - 比分 SP 赔率
   - 胜平负 / 让球胜平负是否支持单关
   - 总进球数赔率
   - 伤病、停赛、可能首发
   - 近期状态
   - 旅途、时差、休整、天气
   - 教练和球员公开采访
4. 把信息分成“确认事实”和“分析推断”。
5. 构建比分单、胜平负单、总进球数单。
6. 输出店主可直接理解的最终订单。
7. 写入当天 Markdown 文件。

### 2. 赛后结算

用户发比赛结果截图后，Skill 会要求 Codex：

1. 识别实际比分。
2. 找到对应日期的 Markdown 记录。
3. 对照当日投注表。
4. 计算每一场的投入、命中、回报和盈亏。
5. 统计当天总投入、总回报、净盈亏。
6. 更新 `今日战果` 和 `今日总结`。
7. 按玩法统计表现：比分、胜平负、总进球数。

## 策略逻辑

这个 Skill 的策略不是“多买几个比分碰运气”，而是更偏向风险投资逻辑：

- 先判断赔率有没有价值。
- 再决定是否下注。
- 控制每天总预算。
- 控制同一场比赛的相关性暴露。
- 赛后持续复盘哪类玩法在亏钱。

## 赔率和期望值

十进制赔率的回本概率线：

```text
回本概率 = 1 / 赔率
```

例如比分赔率是 `6.40`：

```text
1 / 6.40 = 15.625%
```

也就是说，如果你判断这个比分真实发生概率低于 `15.625%`，长期看就不是好下注。

Skill 中使用的期望值门槛：

```text
estimated_probability * odds > 1
```

同时加入安全边际：

- 胜平负 / 让球 / 总进球数：需要大约 `+5%` 优势才算值得。
- 比分：需要大约 `+10%` 优势才算值得，因为比分波动更大。

## 默认资金管理

默认策略是小额、单关、可复盘：

| 项目 | 默认规则 |
|---|---|
| 单注金额 | 2 元 |
| 每日预算 | 24-30 元，常用 28 元 |
| 比分单 | 占 20%-30%，通常 3-4 注 |
| 胜平负 / 让球 | 占 30%-45%，只买支持单关的比赛 |
| 总进球数 | 占 30%-45%，必须和比分逻辑一致 |
| 单场上限 | 通常不超过当天预算 40% |
| 串关 | 默认不串关 |

这套规则的重点是：减少大面积铺比分带来的亏损暴露。

## 为什么不建议大量买比分

比分玩法赔率高，但命中率低，波动很大。

例如一天买 12 个比分，每个 2 元，比分单就已经投入 24 元。即使命中一个低赔率比分，也可能只是回本或小赚；如果不中，亏损会很快放大。

所以 Skill 默认把比分单控制在 3-4 注，把预算转到更稳的方向型玩法和总进球数玩法上。

## 相关性控制

很多投注看似是不同玩法，本质上押的是同一个比赛剧情。

例如：

| 投注 | 隐含剧情 |
|---|---|
| 2:1 | 主队小胜，双方都有进球 |
| 主胜 | 主队不败且赢球 |
| 3 球 | 比赛总进球刚好 3 个 |

如果一场比赛同时大量买 `2:1`、`主胜`、`3球`，看起来是分散，实际上是押同一种走势。

Skill 会尽量避免这种过度相关，只保留最干净的组合。

## 比分模型思路

Skill 不直接实现数学模型代码，但会要求 Codex 用接近 Poisson / Dixon-Coles 的比分分布思维：

- 估计双方进球强度。
- 判断比赛更像低比分、中等比分还是大比分。
- 把比分单和总进球数单联动起来。
- 谨慎处理 `0:0`、`1:1` 这类低比分平局。
- 国家队比赛里，阵容、休整、旅途和战意往往比久远历史数据更重要。

## 新闻、旅途和精神状态

Skill 会要求 Codex 查询公开信息，但必须区分“事实”和“推断”。

可以使用的信息：

- 官方伤停和大名单。
- 教练赛前发布会。
- 球员采访。
- 可靠媒体的训练报道。
- 官方社媒训练图文。
- 航班延误、长途旅行、休整天数、时区变化。

不能做的事：

- 编造球员心理状态。
- 把论坛传言当事实。
- 没有来源就断言航班、内讧、伤病。
- 用“感觉很准”替代赔率和概率判断。

## Markdown 记录格式

每天的记录文件建议包含：

```text
# YYYY-MM-DD 世界杯押注记录

## 今日总览
## 今日订单
## 店主下单内容
## 推荐标记
## 综合分析
## 新闻/旅途/精神状态
## 检查记录
## 今日战果
## 今日总结
## 参考信息
```

订单表建议包含：

```text
场次 | 比分/玩法 | 投注内容 | 金额 | 参考赔率 | 命中回报
```

赛后战果表建议包含：

```text
场次 | 实际赛果 | 命中内容 | 投入 | 回报 | 盈亏 | 备注
```

## 店主订单格式

Skill 会尽量输出这种简单格式：

### 比分单

```text
场次 | 比分 | 金额
荷兰 vs 瑞典 | 2:1 | 2 元
德国 vs 科特迪瓦 | 2:1 | 2 元
合计 | 2 注 | 4 元
```

### 胜平负单

```text
场次 | 玩法 | 投注内容 | 金额
荷兰 vs 瑞典 | 胜平负 | 荷兰胜 | 2 元
荷兰 vs 瑞典 | 胜平负 | 平 | 2 元
合计 |  | 2 注 | 4 元
```

### 总进球数单

```text
场次 | 玩法 | 投注内容 | 金额
荷兰 vs 瑞典 | 总进球数 | 2 球 | 2 元
荷兰 vs 瑞典 | 总进球数 | 3 球 | 2 元
合计 |  | 2 注 | 4 元
```

结尾必须写清楚：

```text
全部单关，不要串关。
```

## 赛后复盘

长期优化靠复盘，不靠临场加码。

建议按玩法统计：

| 玩法 | 统计内容 |
|---|---|
| 比分 | 投入、命中数、回报、ROI |
| 胜平负 / 让球 | 投入、命中数、回报、ROI |
| 总进球数 | 投入、命中数、回报、ROI |

如果连续多天比分单亏损明显，Skill 会倾向于：

- 比分单减少到 1-2 注。
- 把预算转到总进球数和方向型玩法。
- 降低每日总投入。

## 风险提示

- 体育竞猜不是稳定投资。
- 赔率已经包含大量市场信息，个人分析未必有优势。
- 比分玩法尤其高波动。
- 不建议追损。
- 不建议马丁格尔。
- 不建议用生活必要资金参与。
- 请遵守所在地法律法规。
